Project Overview

The goal of this project was to create a tool to assist young Americans in facing one of the top problems in the country. While all of the information is readily available online, searching through spreadsheets of data is not the average American's preferred medium of media consumption in 2024.

The interactive map makes exploring the countries states based on cost of living incredibly easy. Other useful information surrounding the topic of cost according to state is provived, as well as other basic knowledge about the state.

Project Research

I started the project by searching for databases online. Although there is much online about cost of living calculators and generalizations about where it is the cheapest or most expensive to live, most of the data I am using comes from a singular Forbes article. The article contains data sources such as C2ER, KFF, MIT Living Wage Calculator and the U.S. Census Bureau. The calculations they used to determine the cost of living for each state are simple and concise, making applying them to the project much easier.

It is worth mentioning that there are other interactive maps online currently, however they do not display the same data, and are oftentimes embedded in websites with lackluster UI/UX. I wanted to seperate my project from these already existing interactive maps, with pleasing UI/UX and relevant data.

Visual Design

Visual design was an integral part of the process. This aspect is what seperated the project from what already exists. I also personally take much pride in the visual design aspect of the project. For this particular project, I wanted to keep the aesthetic and theme very nuetral. It was meant to appeal to the average American, which could be a lot of different people.

Like most of my designs, I sought to keep the design functional, welcoming, and straightforward. While I believe it is important to create a narrative surrouding the project for audience engagment, I also believe that it is incredibly easy in this day and age for the narrative to do the exact oppposite of its intention. Too much text or required attention for a longer period of time could lose the interest of the user very quickly.

User Interface

The intial idea for my capstone project was actually a compound interest calculator that would demonstrate the power of compound interest in an ineresting way. This idea quickly fell through as I found an interactive map library that was far more interesting than any of the data tables I planned on using prior.

Tackling with D3 library took the most time out of everything in the project. Because of the specific features I had in mind to keep the project interactive, I had to dive deeply into the code of the library. Thanks to much help from professor Mead, Abrar, and chatGPT, we were able to place the files and API inside the project locally, allowing me to directly interact with the libraries contents.

The final project would have three different final slides containing an introduction, form submission, and the interactive map.

  1. Introduction: contains intro to narrative and a few imporatant statistics.
  2. Form Submission: additional engaging narrative and statistics, and a form submission for the user to alter the ineractive map according to their choice.
  3. Interactive Map: displays information on chosen state and chosen submission prior.

Revisions

Most of my revisions were made after the user testing. Thankfully, most of the revisions made were planned for, I just needed user input on a few ideas that I was not sure were as effective as I was projecting them to be.

I covered three main areas for revision:

  1. Navigation: User noted a desire to have some sort of return navigation once on the final slide in order to test the other categories.
  2. Aesthetics: The intial color palette I had chosen could have possible impications of politics, which I had to remove in order to stay nuetral.
  3. Hierarchy of Information: The first order of information I had displayed the final slide was to test which information the user was drawn to first, it was changed accordingly.

Summary

Overall, I believe I was able to achieve what I had set out to accomplish. There some areas where the quality could have used more time to prepare for the final iteration, but I think they are smaller details that only I would notice.When working with libraries in the future, especially from D3, I learned to ask chatGPT for help more than you think is necessary.

Because of different API's or other plug-ins the library might use, attempting to dechipher the code yourself is pointless. Just rely on machine learning. I believe that Professor Mead and anyone who helped me create the project would agree.

For my final digital project at Davis, I think it could have gone a lot worse. Thanks to all who helped.
